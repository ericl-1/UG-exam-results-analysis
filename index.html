+ How This Works What the tool calculates and how it flags questions
How difficult was each question?
The tool calculates what percentage of students got each question right. A question only a quarter of students answered correctly is considered difficult and gets flagged. A question almost everyone got right may be too easy to be useful.
Technically: the difficulty index (p-value) is the proportion of students who answered correctly (0–1). Flagged as difficult when p < 0.35, as too easy when p ≥ 0.96. Scoring uses QM's pre-calculated score column, respecting any post-hoc adjustments.
Did stronger students get it right more often?
For each question, the tool checks whether students who did well overall also tended to get that specific question right. If weaker students did better than stronger ones on a question, something may be off — a confusing wording, a key error, or an unintended hint in the options.
Technically: the discrimination index is the Pearson point-biserial correlation (rpbis) between a student's item score (0/1) and their total exam score. Flagged as poor discrimination when rpbis < 0 and p < 0.80, or as poor disc./difficult when p < 0.25 and rpbis < 0.15.
How consistent is the exam overall?
The reliability score tells you whether the exam as a whole is doing a good job of distinguishing between students of different ability levels. A low reliability score means the exam results may not be very stable — if students took the exam again, their scores might be quite different.
Technically: Cronbach's alpha (α) measures internal consistency. Ranges 0–1. Thresholds: α ≥ 0.80 = Good · α ≥ 0.70 = Acceptable · α < 0.70 = Needs review.
Did English and French students experience the questions the same way?
DIF analysis checks whether any questions performed very differently for English versus French students — beyond what you'd expect from their overall ability levels. A flagged question doesn't mean something is wrong, but it should be reviewed.
Technically: hierarchical logistic regression (Swaminathan & Rogers, 1990). Items flagged when chi-square difference is significant at p < 0.01 (2 df). DIF ≠ bias — it flags items for human review.
Default flagging thresholds
Difficult: p < 0.35 & rpbis > 0  ·  Poor discrimination: rpbis < 0, p < 0.80  ·  Check Key: p < 0.25 & rpbis < 0.15  ·  Too easy: p ≥ 0.96 & rpbis ≥ 0  ·  DIF: p < 0.01 (chi-square, 2 df)  ·  EN/FR gap: amber ≥ 8%, red ≥ 15%
All thresholds are configurable in Settings (hamburger menu).
+ File Preparation Guide What to do — and not do — before uploading
✓ Do this
  • QM: Export the raw results file from QuestionMark as-is (.xlsx)
  • QM: Prepare the answer key file from the same export (see FAQ for steps)
  • Scantron: Use the .txt file received from the scanning centre as-is — no preparation needed
  • Keep all original rows — the tool handles headers automatically
  • QM: Keep the Disclosure question (Q1) — excluded automatically
  • QM: Keep unscored / bloc-note questions — excluded automatically
✗ Avoid this
  • Don't modify, reorder, or delete any rows or columns in either file
  • QM: Don't rename sheets or column headers
  • QM: Don't save as .xls — always use .xlsx
  • QM: Don't combine results and key into a single sheet
  • Don't manually credit or remove questions in the source file — use Question Exceptions in the app instead
  • Don't adjust scores or answers before uploading — let the app handle all calculations
QM stream codes: Default E (English) / F (French). Change in Step 1 if your export uses different codes.
Scantron course codes: Enter the EN and FR course codes used in your exam in Step 1. Students with unrecognised codes can be assigned to a stream at run time.
Upload validation: After uploading, a preview panel appears showing what was detected — check it before running the analysis. Click See details on any status indicator for a full diagnostic.
Question Exceptions: Use the app to credit, delete, or add alternate keys — never modify the source file directly.

About

The Exam Results Analysis Tool automates MCQ item analysis and DIF reporting for UGME exams at the University of Ottawa Faculty of Medicine. Upload a QuestionMark or Scantron export and receive item statistics, score distributions, and formatted Word reports in seconds. For methodology details, see FAQ / Help or How This Works on the setup screen.

Frequently Asked Questions
Current as of  ·  Updated
Getting Started
What files do I need before I can run an analysis? +
It depends on your data source:

QuestionMark (QM): You need two files — (1) the results file (.xlsx), exported directly from QuestionMark, and (2) the answer key file (.xlsx), prepared from the same export (see How do I prepare the answer key file? below). Do not modify either file before uploading.

Scantron: You need a single .txt file received from the scanning centre. No preparation is required — the answer key is embedded in the MASTER row of the file and will be read automatically. Switch the Data Source toggle in Step 1 to Scantron before uploading.
Where do my results files come from? +
QuestionMark: Export your results report directly from QuestionMark as you normally would — no changes to the file are needed before uploading.

Scantron: Your results file is the .txt file received from the scanning centre after the exam is processed. No preparation is required — upload it as-is. The answer key is embedded in the file automatically.
How do I prepare the answer key file? +
Scantron: No preparation needed. The answer key is embedded in the MASTER row of the file received from the scanning centre and is read automatically on upload.

QuestionMark: The answer key comes from the same QuestionMark export as your results file. To prepare it:
  1. In the exported workbook, find the row for the correct answers entry — the one where all responses reflect the answer key rather than a real student's answers
  2. Copy that row to a new sheet within the same workbook (for your records), then copy or move that sheet to a separate new Excel workbook for upload
    • Ensure the column headers from the original file are included — the tool uses these to identify the answer columns
  3. Save and upload that file as your answer key in Step 3
What if my exam has both English and French students? +
The tool handles bilingual exams automatically for both QuestionMark and Scantron. It uses stream codes to separate English and French students and compares their performance in the DIF analysis.

QuestionMark: By default the tool expects E for English and F for French in the results file stream column. If your export uses different codes, update them in the Language Codes field in Step 1 before running the analysis.

Scantron: Students are identified by the course code they entered on their scan sheet. Enter the EN and FR course codes in Step 1 so the tool can match each student to the correct stream. Because students enter these manually, some variation in codes is common — unmatched codes can be assigned to a stream at run time via the correction modal.
Can I use Scantron data instead of QuestionMark? +
Yes. In Step 1 → Data Source, switch the toggle from QM to Scantron. You then upload a single Scantron .txt export — no separate answer key file is needed, as the key is embedded in the MASTER row of the Scantron file. The rest of the analysis works identically to the QM path. When DIF is enabled, enter your EN and FR course codes in Step 1 so the tool can assign each student to the correct language stream.
Some students have unrecognised course codes in my Scantron file — what happens? +
When unrecognised course codes are detected, the upload validation panel will show a warning listing each distinct unmatched code and how many students used it. There are two ways to fix this:

Quick fix (this session): When you click Run Analysis, a correction modal will appear listing each affected student. Assign them to the correct stream using the dropdown, then click Save & Run Analysis. This applies to the current session only.

Global fix: If the mismatch is due to a typo in the EN or FR course code fields in Step 1, correct them there and click Re-validate in the preview panel to refresh the stream counts immediately — no re-upload needed.

Permanent fix: Correct the course codes in the source Scantron file and re-upload.
Pre-Analysis Options
Can I exclude a question before running the analysis? +
Yes — once your files are uploaded and the Run Analysis button is active, a Question Exceptions link will appear just above the button. Click it to open the modal, which lists every question with its answer key letter. For each question you can set the disposition to Include (default), Credit, or Delete. A reason is required for any exclusion. You can also add alternate accepted answers — see below. Click Save Exceptions when done, then run the analysis normally.
What is the difference between Credit and Delete? +
Credit — students who answered correctly keep their point, and the question's point value is removed from the denominator for everyone. This effectively gives every student a small boost. The question's statistics are still shown in the analysis (so you can see how it performed), but it is excluded from flagging and DIF. The question still contributes to the exam average.

Delete — the question is removed entirely. No student receives a point for it, the denominator is reduced by 1 for everyone, and no statistics are calculated. It does not appear in flagging, DIF, or the exam average. It is shown as a greyed row labelled Deleted in the results table.
Can I accept more than one correct answer for a question? +
Yes — use the Alternate Key feature in the Question Exceptions modal. On any question row, click + Alt. Key to expand an inline panel showing answer choices A through E. The original key answer is pre-selected and greyed out for reference. Check any additional letters you want to accept, enter a reason, and click Save.

Students who answered the original key or any alternate answer will receive full credit. All statistics (p-value, discrimination, flagging) recalculate to reflect the updated scoring. The Key column in the results table will show the original answer followed by the alternates in parentheses — for example, A (+D). The reason you entered appears in the Comments column.
Will my exclusions be saved if I re-run the analysis? +
Yes — exclusions are saved in memory for the duration of your session. If you click Update Analysis after changing a setting, your saved exclusions will be applied again automatically. If you click New Analysis or close the browser tab, exclusions are cleared along with everything else. They do not persist between sessions.
Data & Exports
What is the upload validation panel? +
After a successful upload, a parse preview panel appears below the upload fields. It confirms what the tool detected in your files before you run the analysis.

QuestionMark: The panel appears once both the results file and the answer key are uploaded successfully. It shows both filenames, student and question counts, EN/FR stream breakdown, the answer key for the first 10 questions, and a sample of the first 5 student rows. If the question count in the results file doesn't match the key, a mismatch warning appears. If any stream codes are unrecognised, the panel lists each distinct code and how many students used it.

Scantron: The panel appears after the single Scantron file is uploaded. It shows the same information — filename, counts, key preview, and student sample — with unmatched course codes highlighted if any are found.

In both cases this is a preview only — all students and questions are fully loaded regardless. You can fix Step 1 codes and click Re-validate to refresh the stream counts without re-uploading, or click Re-upload if the file itself needs replacing. Click Looks good to dismiss the panel and proceed.
How do I save a record of the analysis for future reference? +
After running an analysis, open the hamburger menu (≡) on the analysis screen and select Save Session Record. This downloads a structured CSV file with three sections: an exam summary row (student counts, averages, reliability, flagged counts), one row per question with all statistics (p-value, discrimination, flag reason, disposition, EN/FR correct rates, DIF result), and the analysis thresholds in effect.

The file is named automatically using the exam title and date. It is designed to be imported into Excel, SharePoint, or other tools for longitudinal tracking. The Elentra ID column is included but blank for now — it will be populated in a future release once item ID linking is resolved.
Can I change the flagging thresholds? +
Yes — open Settings from the hamburger menu. The Analysis Thresholds section has sliders and input fields for all eight configurable thresholds, grouped into MCQ Item Flagging and DIF Analysis. Each group has its own Reset button, and a Reset all to defaults option is available at the bottom.

If you change any threshold after running an analysis, a warning banner will appear prompting you to re-run. The defaults match the UGME manual thresholds — only change them if instructed to do so.
Upload & Setup Issues
I uploaded the file but the tool says it can't read it — what do I do? +
The steps depend on your data source:

QuestionMark: Check that the file is saved as .xlsx (not .xls or .csv). The most common issues are a renamed sheet, a modified header row, or a file re-saved in a different format. Re-export directly from QuestionMark without modifying the file first.

Scantron: Check that the file is a .txt tab-delimited export and that it contains a MASTER row — this is where the tool reads the answer key from. If the MASTER row is missing or the file was exported in the wrong format, re-export from Scantron.

In both cases, click See details on the upload status indicator for a full diagnostic breakdown of what the tool found and what it expected.
The student count looks wrong after uploading — why? +
The tool automatically excludes rows that are not scoreable student responses — this includes header rows, timestamp rows, summary rows, and any students whose stream code doesn't match the EN or FR codes set in Step 1. If the count is unexpectedly low, check that your stream codes in Step 1 match exactly what's in the file (case-sensitive). Click details after uploading for a full row-by-row breakdown.
My stream codes aren't E and F — where do I change them? +
In Step 1 → Language Codes, update the EN and FR fields to match the codes used in your QM export. The tool will re-read the uploaded file using the new codes the next time you run the analysis. Note: changing the codes after an analysis has been run will mark the results as stale and prompt you to re-run.
I uploaded the wrong file — can I replace it without starting over? +
Yes. Simply click the upload area again and select the correct file — it will replace the previous one. If an analysis has already been run, the tool will flag your results as potentially outdated and the Run Analysis button will change to Update Analysis. Re-running the analysis will use the new file.
Running the Analysis
Do I need to enable DIF analysis, or can I skip it? +
DIF analysis is optional. Toggle it on or off in Step 1 → DIF Analysis. When off, the DIF-related tabs, report sections, and KPI indicators are hidden. Turn it on if the exam has both English and French students and you need to identify questions that may have performed differently between streams. If your exam is a single-language cohort, you can leave DIF off.
What happens if I change a setting after running the analysis? +
The tool will detect the change and mark the current results as potentially outdated. An amber warning banner will appear — on the setup page and on the analysis page if you are already there — and the Run Analysis button will change to Update Analysis. Your previous results are still accessible via View stale results, but they reflect the old settings. Re-run the analysis to generate fresh results.

This warning is triggered by: re-uploading a file, changing stream codes, toggling DIF on or off, or changing any analysis threshold in Settings. Changes to the exam title or date do not trigger it — they are cosmetic only.
I need to re-run the analysis with a corrected file — what's the process? +
Click ← Setup on the analysis screen to return to the setup page. Re-upload the corrected file in Step 2 or Step 3 as needed. The Run Analysis button will change to Update Analysis. Click it to re-run — all results, KPIs, and reports will refresh automatically.
Reading the Results
Some questions are flagged — do I need to do anything? +
Your role as a coordinator is to produce the report and pass it along — the exam director reviews flagged items and decides on action (retain, credit, delete, or change the key). The report you export already includes a recommended action for each flagged item. You don't need to make any decisions about individual questions yourself.
The reliability score is showing a warning — should I be concerned? +
A reliability warning (shown in the Reliability KPI) means the exam's internal consistency score (Cronbach's α) fell below the acceptable threshold of 0.70. This is noted in the report automatically. As a coordinator, you don't need to take action — simply include the report as-is. The exam director will interpret this result in context.
There are fewer students in the results than I expected — is something wrong? +
The tool only counts students whose stream code matches the EN or FR codes set in Step 1. Students with a different, blank, or unrecognised stream code are excluded. Check your language codes in Step 1, and click details under the upload confirmation for a diagnostic breakdown showing exactly which rows were included and which were excluded and why.
Exports & Reports
What's the difference between the MCQ Report and the Full Report? +
The MCQ Report covers item-level statistics, flagged questions, and too-easy questions — it's the standard report sent to exam directors. The Full Report combines everything: the MCQ analysis plus the DIF analysis, the Exam Overview section, and all tables in a single Word document. Use the Full Report when the exam director needs everything in one file. The Full Report tab is only available when DIF analysis is enabled.
The Word document looks different on my computer — is that normal? +
Yes, this is expected. The reports are generated using the Calibri font and standard Word styles. If your computer doesn't have Calibri installed, Word will substitute a different font which may slightly affect spacing and layout. The content will always be correct. For best results, open the file in Microsoft Word (rather than Google Docs or LibreOffice).
Can I re-export a report without re-running the analysis? +
Yes. As long as the analysis results are still loaded (you haven't clicked New Analysis or closed the browser tab), you can download the Word or CSV export as many times as you need from the relevant report tab. The analysis stays in memory for your entire session.
Can I save my results and come back to them later? +
Not currently — the tool runs entirely in your browser and does not save session data between visits. If you close the tab or browser, you'll need to re-upload and re-run the analysis. Make sure to download your Word and CSV exports before closing. Session history is planned for a future version.
Troubleshooting
The app froze or showed an error after uploading — what should I do? +
First, try refreshing the page and re-uploading the file. If the error persists, click details under the upload area to see the diagnostic output — this will usually pinpoint the issue (unexpected column headers, missing sheet, incompatible file format, etc.). If you can't resolve it, take a screenshot of the error and the details panel and send it to elarouch@uottawa.ca along with the file you were trying to upload.
Something looks wrong in the report — who do I contact? +
Contact Eric Larouche at elarouch@uottawa.ca. Include the exam name, a description of what looks wrong, and if possible a screenshot or the exported file. The more detail you can provide, the faster the issue can be diagnosed.
